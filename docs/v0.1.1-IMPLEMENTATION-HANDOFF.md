## Implemented and committed

### Installer and CLI

- Root `install.sh` checks Linux and required commands, resolves GitHub's latest release, downloads the versioned archive and `SHA256SUMS`, verifies SHA-256, extracts, and calls the bundled installer.
- `scripts/klyric.ts` implements the requested minimal CLI.
- `scripts/release-download.ts` implements release lookup/download/checksum/extraction for `klyric update`.
- `scripts/local-installation.ts` installs `klyric-bridge`, a `klyric` launcher, and managed CLI runtime under the user's XDG paths; uninstall removes them and preserves settings unless `--purge` is used.
- `scripts/package-release.ts` includes installer/CLI assets.
- `.github/workflows/release.yml` derives version dynamically, validates tag/version agreement, and is prepared to publish archive/checksum assets on a tag build. Publication is still gated.
- Product versions were bumped to `0.1.1` in active package/plugin/plasmoid/bridge/Arch metadata.

### Protocol and Cider state

- Optional protocol-v1 fields `trackHasLyrics?: boolean` and `lyricsPanelOpen?: boolean` were added without breaking old payloads.
- `PlaybackSource` reads guarded MusicKit lyric metadata and reports `.lyric-view-content` presence.
- `PluginStateMachine` carries both fields to published state.
- Protocol TypeScript validation, QML validation, fixtures, and focused tests were updated.

### Widget UX

- `showMusicIcon` setting removed; icon is always visible and alignment-aware.
- Compact display tracks whether content is a lyric; non-lyric content is italic.
- Tooltip uses `Song Name - Artist`.
- Popup is a compact one-to-three-row lyric viewport sized from previous/next toggles.
- Popup diagnostics were removed to keep the requested small lyric-only layout.
